# JWT Authentication API Implementation Plan

**Source spec:** `docs/specs/jwt-auth-api-design.md`

**Goal:** Build a tested Express and TypeScript REST API that persists users in
PostgreSQL and supports registration, login, JWT refresh, and authenticated
profile retrieval.

**Architecture:** Organize the API by feature modules with thin Express
controllers, an authentication service for business rules, a user repository
for Prisma persistence, and shared middleware for validation, authentication,
and error handling. Use stateless access and refresh JWTs with separate
secrets, and run development and isolated test databases on one Docker Compose
PostgreSQL service.

**Tech stack:** Node.js 24, npm, Express, TypeScript ESM, Prisma ORM 7,
PostgreSQL, `@prisma/adapter-pg`, Zod, bcrypt, jsonwebtoken, Vitest, Supertest,
ESLint, Prettier, and Docker Compose.

---

## Execution Prerequisites

- Node.js 24 and npm are installed. The planning environment currently reports
  Node `v24.13.1` and npm `11.8.0`.
- Docker Compose must be available before database and integration-test tasks.
  Docker is not currently available inside this WSL environment; enable Docker
  Desktop WSL integration or provide an equivalent Docker engine before
  executing Tasks 2 and later database verification.
- Run commands from the repository root.
- Dependency installation commands use compatible current releases while
  explicitly pinning Prisma packages to major version 7 because the plan uses
  Prisma 7 APIs. `package-lock.json` created in Task 1 becomes the exact
  dependency source of truth for subsequent execution.
